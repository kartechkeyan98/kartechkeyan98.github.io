Multi-Layered Perceptron (MLP)

The Multi-Layered Perceptron (MLP) is the original Neural Network that flooded the market. This is the most basic topology in deep learning.

Things to know about any NN topologies in general:

  1. Any NN topology is by design meant to take in one specific type of data, from an input space X, transform it into another type of data that resides in the output space Y.

  2. The logic behind the transformation is what governs the equation of the forward pass [process of feeding input, getting output]. This is what is to be understood. The equations themselves are just embodiment of the design philosophy.

  3. Almost every NN topology can be broken down into layers which all come from a finite set of blueprints, just with different hyperparameters. For example, an MLP is basically a sequence of Linear + Activation layers, one unit (with different hyperparameters) repeating again and again.

The input space for an MLP is: Rdi​, i.e. space of real vectors, and the output space is some manifold, which lives in Rdo​, again a space of real vectors.

The point of a neural network is to capture or approximate any continuous vector valued function by tuning the parameters of the model from the data.

1. Basic Units of MLP

The basic units of an MLP are two layers, Linear & Activation layer. Each layer comes with two operations.

  1. Forward Pass: The actual transformation of input to output, y=f(x;θ).
  2. Backward Pass: Taking the gradient wrt output of a loss function, δy​ and computing:
    • Gradient wrt Parameters, δθ​
    • Gradient wrt Input, δx​

These two operations are present in every single layer, and by composition, every single NN topology. The purpose of layering is explained in the following subsection.

1.1. Loss Function

Before moving forward, the goal of any DL architecture is to provide a model blueprint which is able to learn a dataset D={(x(i),y(i))} efficiently.

The meaning of that is:

1. For all x(i)∈D0​ ={x(i)}, the output of the network y^​(i)=f(x(i);θ) should be close to the true value y(i)∈D1​ ={y(i)}. Some function measuring the closeness between two values in Y is required: l(⋅,⋅). As l(y^​(i),y(i))→0, closer the predictions on the training dataset are. The thing to be minimized is:

L(θ;D)≜∣D∣1​i∑​l(y^​(i),y(i))≡∣D∣1​i∑​l(i)

This is straight from the Empirical Risk Minimization framework, where l(⋅,⋅) serves as a measure of goodness of responses of the model on a single datapoint. L(θ;D) depends on (a) the parameters, (b) the dataset itself, which is assumed fixed.

Hence, the goal becomes:

θ∗=argθmin​L(θ;D)

2. The standard optimization methods like SGD, ADAM, RMSprop etc. use only first order information, viz the gradient of L(θ) wrt θ to update the parameters in a way that iteratively minimizes the Empirical Risk. Hence, to calculate ∇θ​L(θ). The efficiency part comes from here in the sense that, the algorithm to find the optimal θ∗ should be low in complexity.

3. Usually, transforms are chained in sequence, which translates to composing transforms. This is called layering, instead of evaluating in one go, the computation occurs in small steps. Only after the last layer will the loss function be evaluated. The concept is illustrated as follows (for one single datapoint):

​z0​z1​z2​...zL​lx​​​=x=f1​(z0​;θ1​)=f2​(z1​;θ2​)=fL​(zL−1​;θL​)=l(zL​,y)​

This is called a composition of transformations each with their own parameters θi​ i=1,2,...,L

The goal is to calculate gi​≡∇θi​​lx​ for all i=1,2,...,L. Suppose magically you obtain ∇zk​​lx​≡δzk​.

dzk​​=j∑​∂zk−1,j​∂fK​​dzk−1,j​+m∑​∂θk,m​∂fK​​dθk,m​→Total Derivitive Theorem=JfK​​(zk−1​)dzk−1​+JfK​​(θk​)dθk​​∴dlx​​=δzk⊤​dzk​→Total Derivitive Theorem=δzk⊤​JfK​​(zk−1​)dzk−1​+δzk⊤​JfK​​(θk​)dθk​=δzk−1⊤​dzk−1​+gk⊤​dθk​​∴​δzk−1​=JfK​​(zk−1​)⊤δzk​δθk​≡gk​=JθK​​(θk​)⊤δk​​

The last part is reexpressing total derivitive theorem in terms of zk−1​ & θk​

Hence, in one fell swoop, from δzk​, extracted δzk−1​ & ∇θi​​lx​, provided dependence of zk​ on zk−1​ & θk​ is known. Now one of those can be used to calculate the gradients up the chain. This is a recursive algo, which can be made iterative with a few clever tricks.

This recursive process of calculating derivitives is called backpropagation aka backprop or Backward Pass. The only thing missing is the base case, which is δzL​ usually hardcoded in autograd engines.

NOTE

This might as well be the most important part of this chapter. It is not the formulas, but the idea of using total derivitive theorem like this that is the star of the show.

This technique will be used again and again in many of the following chapters, getting this down at the start itself is better than struggling later.

1.2. Linear Layer

Forward Pass​

This is a fancy name for an affine linear transformation of the input vector.

Definition 1.1 [Forward Pass of Linear Layer]

x∈Rdi​, [di​-dim. vector] is the input. The Linear Layer transformation (aka Forward Pass) is:

​1. Forward Pass​: y=Wx+b​Where W∈Rdo​×di​ and b∈Rdo​are the parameters of this layer, theWeight Matrix and Bias Vector2. Time Complexity​: O(do​⋅di​)​For a single input, which comes from onematrix-vector product and vector addition.​

Geometrically, on the whole input space, this can be viewed as transforming the whole of Rdi​ into an affine subspace, which is b-shifted span of columns of W.

Backward Pass​

Given to you is ∂y∂l​ ≡δy​∈Rdo​.

The goal for the backpass of a linear layer is: Given δy​, compute ∂W∂l​≡ δW​∈Rdo​×di​, ∂b∂l​≡ δb​∈Rdo​ & ∂x∂l​≡δx​∈Rdi​. The first two are the parameter gradients, the last one is input gradient which will be passed to upstream layers.

Theorem 1.1 [Backpass of Linear Layer]

Linear Layer forward pass: y=Wx+b

  1. Given δy​∈Rdo​ (output gradient),

​1. δb​∈Rdo​=δy​2. δW​∈Rdo​×di​=δy​⋅x⊤3. δx​∈Rdi​=W⊤⋅δy​​​

  1. The total time complexity of the backpass is O(2⋅do​⋅di​)
Proof
1.1.1 The standard proof

​yi​=j∑​Wij​xj​+bi​​​⟶Def. 1.1.1, Linear Layer Fwd. Pass​​Observe that bi​ only affects yi​. Hence,​∴​∂bi​∂l​≡δb,i​=∂yi​∂l​⋅∂bi​∂yi​​=δy,i​δb​=δy​​

  • Much more elegent Matrix Calculus proof

∴​y=Wx+bdo​×1∂b∂l​​=do​×do​∂b∂y​​⋅do​×1∂y∂l​​​​⟶Def. 1.1.1, Linear Layer Fwd. Pass⟶ Chain rule ​Where, ∂b∂y​∈Rdo​×do​=[∂bj​∂yi​​](do​×do​)=[δij​]do​×do​​=Ido​​since bi​ only affects yi​, as given in forward pass,the off diagonal elements are all 0 and on diag.elements are 1.∴​δb​=Ido​​⋅δy​=δy​​q.e.d

1.1.2  Standard Proof

​yi​=j∑​Wij​xj​+bi​​​⟶ From def. of fwd pass​​Observe that Wij​ only affects yi​. Hence,​∂Wij​∂l​≡δW​[i,j]=∂Wij​∂yi​​⋅∂yi​∂l​=xj​δy,i​​​This is the formula for an outer product:δW​=δy​⋅x⊤​

  • Matrix Calculus

∴∴​y=Wx+bdy=dW⋅x+W⋅dx+dbdl=δy⊤​dy​​⟶Total Derivative​Since we need to calculate the effect of change in W to yand hence l, we shut off dx & db.∴∴∴∴​dl=δy⊤​⋅dW⋅xdl=tr(δy⊤​⋅dW⋅x)dl=tr(x⋅δy⊤​⋅dW)dl=tr(δW⊤​⋅dW)​​⟶ Cyclic Trace Identity​Hence, δW​=δy​⋅x⊤ q.e.d

1.1.3  Now just using matrix calculus. It is much more elegant and less cumber some to write.

∴​y=W⋅x+bdy=dW⋅x+W⋅dx+db​Since we are concerned with how x↦x+dxaffects y hence l, shut off all the other differentials.∴∴∴​dl=δy⊤​⋅dydl=δy⊤​⋅(W⋅dx)dl=δx⊤​⋅dx​∴δx​=W⊤δy​ q.e.d

1.2  Observe, in the three calculations above, only the second and third involve computing something new, and they are:

  • Outer product of (do​,),(di​,) which is O(do​⋅di​).
  • Matrix Vector Product of (di​,do​),(do​,1) which is O(do​⋅di​)

Hence, in total O(2⋅do​⋅di​). q.e.d

Proofs 1.1.1 & 1.1.2 show some nice tricks using matrix calculus to find derivatives. Neat'o. The trick is simple, if you need to find ∂h∂l​:

  • h is scalar, express dl as g⋅dh, δh​=g.

  • h is vector, express dl as g⊤dh, δh​=g.

  • h is matrix, express dl as tr(G⊤dh), δh​=G.

This is the crux of the whole matrix differentiation business. Notice that these are all some sort of inner products. For higher ranked tensors, express dl as a dot product of dh with something, say G. Then δh​=G.

Even in case of matrix, that trace equation, all you get is total derivative theorem packed in a convenient way with known matrix operations. Same for vector.

1.3 Activation Layer

If we just use linear layer, y=W⋅x+b, notice that all we are then doing is modeling the input output relation as a linear one. This is not always true for real world data, which is more often than not, nonlinear and very complex. Even C1 may not always be a valid assumption.

This is where activation functions come in, which are elementwise functions, that introduce nonlinearity into the mix. The general activation function is a nonlinear function ϕ:R↦R, which is applied elementwise to the output vector of the linear layer.

The activation functions are usually continuous, hence for a significant portion of the architectures, C1 relation between input and output is assumed, even though, in real life, it need not be like that.

Forward Pass​

Definition 1.2 [Activation Layer Forward Pass]

Given ϕ:R↦R, an activation function,

1. Forward Pass​: y=ϕ(x)∈Rd​

Forward pass on a vector x∈Rd
Where yi​=ϕ(xi​)∀i=0,1,...,d−1 (elementwise operation)

2. Time Complexity​: O(d)

Operation on x∈Rd has d elementwise calculations.

Backward Pass​

Theorem 1.2 [Activation Layer Backpass]

Activation Layer Forward Pass: y=ϕ(x)

  1. Given δy​∈Rd (output gradient)

δx​∈Rd=δy​⊙ϕ′(x)​​

  1. The time complexity is O(d)
Proof
1.1  Given δy​=∂y∂l​,

∴​y=ϕ(x)dy=ϕ′(x)⊙dx​​⟶Elementwise differential⟶(s1)​∴∴​dl=δy⊤​dydl=δy⊤​(ϕ′(x)⊙dx)​​⟶Total Derivative⟶ from (s1)​∴∴∴∴​dl=δy⊤​⋅diag(ϕ′(x))⋅dxdl=δx⊤​⋅dxδx​=diag(ϕ′(x))⋅δy​δx​=ϕ′(x)⊙δy​​​→Convert to Matmul form→Total Derivative→Diagonal matrix is its own transpose→Converting to Hadamard Product​ q.e.d

1.2 Since the calculation involves d derivative calculations and scalar products, time complexity is O(d)q.e.d

2. MLP Operations

This section deals with chaining Linear + Activation layer units, hence creating deep networks.

This is the actual MLP, which is a chain of different Linear + Activation transforms stacked together.

2.1 Forward Pass

Suppose you have multiple stacks of linear + activation layers, which is a composition of transforms.

⋮​Input​:a0​=xHidden Layer 1​:z1​=f1​(a0​)=W1​⋅a0​+b1​a1​=ϕ1​(z1​)Hidden Layer 2​:z2​=f2​(a1​)=W2​⋅a1​+b2​a2​=ϕ2​(z2​)Hidden Layer L​:zL​=fL​(aL−1​)=WL​⋅aL−1​+bL​aL​=ϕL​(zL​)​

One linear + activation layer is called a hidden layer. This sequence of computations is what is called an MLP. This can be written in the form of an algorithm, which is the Forward Pass for an MLP.

In essense, this is the repeated application of definition 1.1 & 1.2.

Algorithm 1.1 [MLP Forward Pass]

Input​:

  • x∈Rd0​, input to the network.
  • θ={Wi​,bi​}i=1L​, the parameters of the network.

Output​:

  • y∈RdL​, output of the network.

Algorithm​:

1. a0​=x# ∈Rd0​2. for i=1,2,…L do​# Definition 1.1.1, Linear Layer forward pass1. zi​=Wi​⋅ai−1​+bi​∈Rdi​# Definition 1.2.1, Activation layer forward pass2. ai​=ϕi​(zi​)∈Rdi​​end for 3. return aL​

Given the output size of each layer is: d0​,d1​,d2​,…dL​, the time complexity for the forward pass of a single input is: O(∑i=0L−1​di​⋅di+1​), from definition 1.1.2 & definition 1.2.2.

2.2 Backward Pass

This section is the repeated application of theorem 1.1 & 1.2. From these, it is evident that for all linear + activation units, two things are required to be stored: ai−1​ & zi​, i.e. inputs to each linear and activation layers are required for the backpass.

A small tidbit, the ai​'s are called activations and zi​'s are called preactivations in MLP parlance.

Algorithm 1.2 [MLP Backpass]

Input​:

  • θ={Wi​,bi​}i=1L​, network parameters.
  • δaL​≡∂al​∂l​ ∈RdL​, gradient wrt network output.
  • A={ai​}i=0L−1​ and Z={zi​}i=1L​, the relevent activations and preactivations respectively.
    If not cached during forward pass, recalculate it.

Output​:

  • δWi​= ∂Wi​∂l​ ∈Rdi​×di−1​, i=1,2,…L
  • δbi​= ∂bi​∂l​ ∈Rdi​, i=1,2,…L
  • δx​=∂x∂l​ ∈Rd0​

Algorithm​:

# Optional Step if values not cached0. Calculate {zi​}i=1L​,{ai​}i=0L​ using algorithm 1.1 1. for i=L,L−1,…1 do​# Theorem 1.2.1, Activation layer backpass1. δzi​≡∂zi​∂l​=δai​⊙ϕi′​(zi​)∈Rdi​# Theorem 1.1.1, Linear layer backpass2. δbi​≡∂bi​∂l​=δzi​∈Rdi​3. δWi​≡∂Wi​∂l​=δzi​⋅ai−1⊤​∈Rdi​×di−1​4. δai−1​=Wi⊤​⋅δzi​∈Rdi−1​​end for2. return {δWi​,δbi​}i=1L​,δx​≡δa0​

The input gradient is required in case that itself is coming from some upstream layer, hence outside the scope of this algo, continuing the backprop upstream.

The time complexity of backprop on whole MLP is: O(∑i=0L−1​di​⋅di+1​) from theorem 1.1.2 & theorem 1.2.2.

3. Batching in MLP

The previous section detailed the two fundamental operations of any NN, forward & backward pass for MLP with single input. But the dataset D has N datapoints.

One option is, you can loop over all the datapoints and calculate the outputs, then gradients and proceed, but from an implementation PoV, it would become very tedious and would not utilize the hardware fully.

The interesting thing to note is, multiple forward passes through linear and activation layer can be converted into a single matrix multiplication.

Definition 1.3 [Batched Forward Pass]

1. Linear Layer​

Given input vectors x(i)∈Rd0​'s in the form of a matrix X∈RN×d0​ as follows:

X=​−−−​x(0)⊤x(1)⊤⋮x(N−1)⊤​−−−​​∈RN×d0​

and the parameters for the linear layer being W∈Rd1​×d0​ & b∈Rd1​ as usual, the forward pass for a whole batch is:

Y=X⋅W⊤+b​b⊤∈RN×d1​​

+b​ is broadcasted addition, and the time complexity of this operation is
O(N⋅i=0∑L−1​di​⋅di+1​)​

2. Activation Layer​

Given input vectors x(i)∈Rd0​'s in the form of a matrix X∈RN×d0​ as follows:

X=​−−−​x(0)⊤x(1)⊤⋮x(N−1)⊤​−−−​​∈RN×d0​

and ϕ being the elementwise activation function, the forward pass in batch becomes: Y=ϕ(X)∈RN×d0​​

The time complexity of this operation is
O(N⋅d0​)​

Notice that the time complexity is the same as that of looping over datapoints individually. But current hardware is extremely efficient in performing matrix multiplications, which makes this method much more efficient for the linear layer and trivially parallelzable for activation layer.

3.1 Batched Backprop

Remember from Section 1.1 that the Empirical Risk for the whole dataset can be written as:

​L(θ;D)=∣D∣1​⋅i=0∑N−1​l(y^​(i),y(i))y^​(i)≡f(x(i);θ)​

where f(x;θ) is the MLP model, whose parameters θ is to be optimized such that

θ∗=argθmin​L(θ;D)

i.e. Empirical Risk Minimization (ERM).

Again, for using first order methods like SGD, RMSProp, ADAM etc, ∇θ​L(θ;D) has to be calculated. By linearity of the ∇ operator:

∇θ​L(θ;D)=∣D∣1​⋅i=0∑N−1​∇θ​l(y^​(i),y(i))

∇θ​l(y^​(i),y(i)) is calculated using backpropagation for a single datapoint.

Brute forcing this, the naive algorithm would be:

Naive Batched Backpass

  1. g=0, is the accumulator of gradients.
  2. for i=0,1,⋯N−1 do
    1. y^​(i)=f(x(i);θ), forward pass using Algorithm 1.1 in Section 2.1.
    2. l(i)=l(y^​(i),y(i)), calculate loss for one datapoint.
    3. Calculate δθ(i)​ i.e. gradients for i-th datapoint using Algorithm 1.2 in Section 2.2.
    4. g+=δθ(i)​
  3. end for
  4. return g/∣D∣

Towards a better, more efficient way of calculating backpass in terms of matrix multiplications:

The overall procedure of backpropagation remains the same: find output gradients first, which in batched case is AL​∈RN×dL​, so the corresponding gradient is: δAL​ ≡∇AL​​L(θ;D).

∴​AL​=​−−−​aL(0)⊤​aL(1)⊤​⋮aL(N−1)⊤​​−−−​​∈RN×dL​L(θ;D)=∣D∣1​⋅i=0∑N−1​l(y^​(i),y(i))=∣D∣1​⋅i=0∑N−1​l(aL(i)​,y(i))∇AL​​L(θ;D)=∣D∣1​⋅i=0∑N−1​∇AL​​l(aL(i)​,y(i))∇AL​​L(θ;D)=∣D∣1​⋅i=0∑N−1​​−−−​∇aL(0)​​l(aL(i)​,y(i))⊤∇aL(1)​​l(aL(i)​,y(i))⊤⋮∇aL(N−1)​​l(aL(i)​,y(i))⊤​−−−​​∈RN×dL​​

The last step comes about by expanding the gradient. This is where the magic happens since:

∇aL(j)​​l(aL(i)​,y(i))={0∇aL(i)​​l(aL(i)​,y(i))​i=ji=j​

effectively, for i-th matrix in summation, only its i-th row is non-zero, rest all rows are zeroed out. This simplifies the calculations:

∇AL​​L(θ;D)​=∣D∣1​⋅​−−−​∇aL(0)​​l(aL(0)​,y(0))⊤∇aL(1)​​l(aL(1)​,y(1))⊤⋮∇aL(N−1)​​l(aL(N−1)​,y(N−1))⊤​−−−​​∈RN×dL​=∣D∣1​⋅​−−−​∇aL(0)​​l(0)⊤∇aL(1)​​l(1)⊤⋮∇aL(N−1)​​l(N−1)⊤​−−−​​​​

Each row is an output gradient given for an individual backpass.

KEY TRICK

This is not just true for AL​, it is true for any Al​, Zl​, i.e each row of the gradient matrix is the gradient from an individual backpass, which is why the gradient matrices will have N rows for activations, preactivations and input.

This happens since the computations due to zl(j)​ cannot affect l(aL(i)​,y(i)), i.e computations due to the j-th datapoint cannot affect the loss of the i-th datapoint, so any variable appearing in the computation sequence of j-th datapoint like zL(j)​ or aL(j)​ will not affect the i-th loss l(i).

This follows from the implicit assumption of ERM, which states that datapoints are independent and identically distributed (i.i.d).

This is the key trick underneath all the mathematics. The only catch is: the output gradient matrix must be given or should be calculable somehow.

This leads to better version of batched backpass. First going layer wise:

Theorem 1.3 [Batched Backprop of Linear & Activation Layer]

1. Linear Layer​

Given Y=X⋅W⊤+b​b⊤ the forward pass for linear layer [Definition 1.3.1], and δY​=∇Y​L(θ;D)∈RN×d1​:

δY​=∣D∣1​​−−−​δy(0)⊤​δy(1)⊤​⋮δy(N−1)⊤​​−−−​​=∣D∣1​​−−−​∇y(0)​l(0)⊤∇y(1)​l(1)⊤⋮∇y(N−1)​l(N−1)⊤​−−−​​

Then,

​1. δb​=sum(δY​,axis = 0)∈Rd1​2. δW​=δY⊤​⋅X∈Rd1​×d0​3. δX​=δY​⋅W∈RN×d0​​​

The sum operation denotes a row sum (axis = 0).

2. Activation Layer​

Given Y=ϕ(X) from [Definition 1.3.2] and δY​ =∇Y​L(θ;D) ∈RN×d1​,

δX​=δY​⊙ϕ′(X)∈RN×d1​​

Proof
1. Linear Layer​
Y=X⋅W⊤+b​b⊤⟶ Definition 1.3.1, Batched Forward Pass, Linear LayerY=X⋅W⊤+1N​⋅b⊤⟶Rewrite as an outer product∴dY=dX⋅W⊤+X⋅dW⊤+1N​⋅db⊤∴​dL=tr(δY⊤​dY)=tr(δY⊤​⋅dX⋅W⊤)+tr(δY⊤​⋅X⋅dW⊤)+tr(δY⊤​⋅1N​⋅db⊤)=tr(W⊤⋅δY⊤​⋅dX)+tr(δY⊤​⋅X⋅dW⊤)+tr(δY⊤​⋅1N​⋅db⊤)⟶Cyclic property of trace=tr(W⊤⋅δY⊤​⋅dX)+tr(dW⋅X⊤⋅δY​)+tr(δY⊤​⋅1N​⋅db⊤)⟶tr(A⋅B)=tr(B⊤⋅A⊤)=tr(W⊤⋅δY⊤​⋅dX)+tr(X⊤⋅δY​⋅dW)+tr(δY⊤​⋅1N​⋅db⊤)⟶Cyclic property of trace=tr(W⊤⋅δY⊤​⋅dX)+tr(X⊤⋅δY​⋅dW)+tr(db⋅1N⊤​⋅δY​)⟶tr(A⋅B)=tr(B⊤⋅A⊤)=tr(W⊤⋅δY⊤​⋅dX)+tr(X⊤⋅δY​⋅dW)+tr(1N⊤​⋅δY​⋅db)⟶Cyclic property of trace​

Hence, by total derivitive theorem for matrices:

​δb​=δY⊤​1N​=sum(δY​,axis = 0)δW​=δY⊤​XδX​=δY​⋅Wq.e.d​

2. Activation Layer​

3.2 Batched MLP Algorithms
